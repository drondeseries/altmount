# Getting Started with AltMount

AltMount is a high-performance WebDAV server backed by Usenet that provides seamless streaming and integration with media management tools like Sonarr, Radarr, and other ARR applications.

## What is AltMount?

- **WebDAV Interface** -- Mount NZB content as a standard filesystem
- **Streaming** -- Stream media directly from Usenet without full downloads
- **Rar/7zip Support** -- Transparent extraction with full support for:
  - Multi-part RAR archives (`.rar`, `.r00`–`.r99`, `.part01.rar`, …)
  - Multi-part 7-zip archives (`.7z`, `.7z.001`, `.7z.002`, …)
  - RAR archives nested inside other RAR archives
  - RAR archives nested inside 7-zip archives
  - Password-protected RAR and 7-zip archives
  - Blu-ray ISO images (`.iso`)
- **ARR Integration** -- Native Sonarr/Radarr integration via SABnzbd-compatible API
- **Symlink/STRM Support** -- Category-based symlinks or STRM files for cleaner organization
- **Health Monitoring** -- Automatic corruption detection and repair
- **Web Interface** -- Modern UI for configuration and monitoring

## Quick Start

### 1. Run with Docker Compose

Create a `docker-compose.yml` file:

```yaml
services:
  altmount:
    extra_hosts:
      - "host.docker.internal:host-gateway"
    image: ghcr.io/javi11/altmount:latest
    container_name: altmount
    environment:
      - PUID=1000
      - PGID=1000
      - PORT=8080
      - JWT_SECRET=change-me-to-a-strong-random-secret # Required: used to sign JWT tokens
      - COOKIE_DOMAIN=localhost
    volumes:
      - ./config:/config
      - /mnt:/mnt:rshared
    ports:
      - "8080:8080"
    restart: unless-stopped
    devices:
      - /dev/fuse:/dev/fuse:rwm
    cap_add:
      - SYS_ADMIN
    security_opt:
      - apparmor:unconfined
```

Start the service:

```bash
mkdir -p ./config/metadata
docker-compose up -d
```

> `JWT_SECRET` is **required when login is enabled** (`auth.login_required: true` in your config). Set it to a strong random string (e.g. `openssl rand -hex 32`). The server will not start without it if login is enabled.
>
> Set `COOKIE_DOMAIN` to match how you access the web interface (e.g., `localhost`, `192.168.1.100`, or `altmount.example.com`).

### 2. Register

Open `http://localhost:8080` in your browser. On first access, create an admin account.

![User registration screen](/images/register.png)

### 3. Add NNTP Providers

Navigate to **Configuration > Providers** and click **Add Provider**. Enter your Usenet provider details: host, port, credentials, max connections, and SSL settings.

![Provider configuration](/images/config-providers.png)

### 4. Select Import Strategy

Navigate to **Configuration > Import** and choose how downloaded files are made available:

- **NONE** -- Use the WebDAV mount directly (simplest option)
- **SYMLINK** -- Create category-based symbolic links for easier access
- **STRM** -- Generate STRM streaming files for Emby/Jellyfin

![Import strategy configuration](/images/config-import.png)

### 5. Select Mount Type

Navigate to **Configuration > Mount** and choose your mount method:

- **FUSE** -- Built-in FUSE mount (Linux containers only)
- **RClone** -- Built-in rclone mount with VFS caching
- **None** -- Use an external mount tool on your host

![Mount type configuration](/images/config-mount.png)

### 6. Configure ARR Integration (Optional)

If you use Sonarr/Radarr, navigate to **Configuration > ARRs** to add your instances. AltMount acts as a SABnzbd-compatible download client.

![ARR integration settings](/images/config-arrs.png)

You'll also need to configure the SABnzbd API under **Configuration > SABnzbd** with categories matching your ARR setup (e.g., `tv` for Sonarr, `movies` for Radarr).

![SABnzbd configuration](/images/config-sabnzbd.png)
